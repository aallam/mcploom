# Next Release Checklist (2026-03-28)

This release should be published manually from the release PR branch.

Why this is the simplest bootstrap:

- the first-release packages must not be first-published from GitHub Actions
- the already-published packages need the version bumps generated by Changesets
- publishing all packages manually from the release PR branch gives one exact release state
- after that, CI sees those versions already on npm and skips them

## Required sequence

1. Merge the release-prep PR that adds the Changesets files and CI wrapper.
2. Wait for the Changesets GitHub Action to open or update the release PR.
3. Check out the release PR branch locally.
4. Build and test from that exact branch.
5. Publish all public packages manually from that exact branch.
6. Verify the published versions on npm.
7. Merge the release PR.
8. Let GitHub Actions run and skip the already-published versions.

## Manual publish preflight

Run these checks from the release PR branch:

```bash
npm whoami
npm ci
npm run build
npm test
```

## CI behavior after manual publish

The publish workflow uses `node scripts/changeset-publish.mjs` instead of calling `changeset publish` directly.

That wrapper:

- refuses to let GitHub Actions first-publish the new codexec packages
- allows a normal post-merge no-op once all release versions already exist on npm
- keeps future releases on the normal Changesets CI path after this bootstrap release

## Failure recovery

If the release PR is merged before all packages are manually published:

1. finish publishing the missing packages from the release PR state
2. re-run the failed publish workflow
